# rstTools README

Сборник инструментов для работы с разметкой reStructuredText.

## Возможности

Какие инструменты доступны:

1. Управление переменными:
   
   - Подставляет инлайновые инклюды (переменные) из файлов, которые подключены в файле `conf.py` проекта или определены в самом файле. 
     Начните вводить `|` и значение переменной для отображения списка доступных переменных.

   - Показывает значение переменной и ее расположение при наведении на нее.

   - Подсвечивает переменные, которые не удалось определить.

   - Позволяет перейти к редактированию переменной.
     Для этого нужно кликнуть на переменную с зажатой клавишей `Ctrl`.
   
   - Позволяет открыть в новой вкладке картинку, путь к которой указан в переменной.
 
2. Управление ссылками внутри документа:

   - Позволяет подставлять относительные пути к файлам проекта внутри роли `doc`.
     Начните вводить путь для отображения доступных файлов.

   - Позволяет подставлять пути к файлам другого проекта внутри роли `doc`.
     Начните вводить название проекта для отображения доступных проектов.
   
   - Показывает путь к файлу в ссылке и заголовок этого файла при наведении на ссылку.

   - Подсвечивает некорректные ссылки, в которых путь к файлу указан неправильно.

   - Подсвечивает некорректные ссылки на другие проекты, если проект не найден в директории `source/ru/ru`.

   - Позволяет открыть в новой вкладке файл, путь к которому указан в ссылке.

3. Управление инклюдами:
   
   - Показывает расположение файла с инклюдом при наведении на него.

   - Позволяет открыть в новой вкладке файл инклюда и сразу ставит курсор к нужному фрагменту.
   
   - Подсвечивает некорректные инклюды, в которых путь к файлу указан неправильно.

   - Подсвечивает некорректные инклюды, в которых начало или конец вставляемого фрагмента указаны неправильно.

## Release Notes

Список изменений.

### 1.0.0

Первая версия расширения. Включает мастер управления переменными и мастер управления ссылками внутри документа.

### 1.1.0

Поддержаны переменные для изображений.

### 1.2.0

Поддержаны инклюды.

### 1.2.3

Инклюды и файлы в `doc` открываются по ссылке.

### 1.2.4

Улучшена проверка путей в ссылках и инклюдах.

### 1.3.0

- Добавлено быстрое действие для ошибки "Проект не подключен в intersphinx_mapping". Оно позволяет добавить проект сразу в `conf.py` нужного документа.
- При открытии файлов с инклюдами через Ctrl + Click теперь учитывается контекст. Ссылки и переменные после открытия файла проверяются относительно расположения родительского файла.
- Добавлен предпросмотр фрагмента инклюда при наведении на путь к файлу.
- Улучшена фильтрация файлов и каталогов при вводе пути внутри роли `doc`.

### 1.3.1

- Рефакторинг: повторяющиеся вспомогательные функции вынесены в общие модули `utils/contextResolver`, `utils/workspaceResolver`, `doc/docUtils`.
- Межпроектные ссылки теперь не требуют указания проекта в `intersphinx_mapping`. Доступны все проекты из директории `source/ru/ru`.
- Добавлена проверка, что цель ссылки `doc` является файлом, а не директорией.
- Ошибочные ссылки и переменные подчёркиваются полностью, включая символы `|` и `` ` ``.
- Сообщения об ошибках больше не дублируются в тексте ховера — они отображаются только в подсказке диагностики.
